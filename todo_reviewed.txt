2. Minimal-Compiler v1
============================================================

Ziel:
    Ein kleines, konsistentes Ende-zu-Ende-Subset, das von .ry ueber AST und IR
    bis Assembly laeuft.

Parser / AST:
    - Eigene Block-Scopes mit { ... } als Statement sauber abbilden.
    - Optional: bessere Fehlertypen mit Tokenpositionen/Spans.

IR-Codegen:
    - BinaryOps fuer int/bool/float lowern:
        + - * / %
        == != < <= > >=
        and/or
        & | ^ << >>
    - UnaryOps lowern:
        - ! ~
    - Assignments lowern:
        x = expr
        x += expr
        x -= expr
        x *= expr
        x /= expr
    - Expr-Statements lowern.
    - Function calls lowern:
        normale Calls
        Rueckgabewerte
        void-Calls
        Argumentanzahl und Argumenttypen pruefen
    - Mehrere Basic Blocks erzeugen:
        if / else / else if
        while
        loop
        break
        continue
    - Scope-Stack komplett nutzen:
        Function-Scope
        Block-Scope
        if/while/loop/for-Scope
        globaler Scope
    - Char-Literals lowern.
    - Null-Literals typisieren oder vorerst gezielt verbieten.
    - Globals und consts fuer einfache primitive Konstanten erzeugen.
    - Funktionsnamen eindeutig machen.
        Noch klaeren:
            Was genau wird gemangelt?
            Nur Overloads?
            Methoden?
            Generics/Traits spaeter?

IR-Design / Typtracking:
    - Jede TempId braucht einen bekannten IrType.
    - TempIds aus Alloca/GlobalAddr/GetFieldAddr/GetElementAddr sind Adressen:
        Pointer(inner_type)
    - Jeder IrType::Named(name) muss genau eine passende IrTypeDefinition
      in IrModule.types haben.
    - Pointer-Typen fuer alloc/load/store/globaladdr/getfieldaddr/getelementaddr
      konsequent verwenden.

IR to Assembly:
    - Calls und Parameter gegen das gewaehlte Calling-Convention-Modell pruefen.
    - Stack-Alignment fuer Calls absichern.
    - Bool-Operationen auf 0/1 normalisieren.
    - Strings noch nicht implizit als Wert vortaeuschen, solange String-Layout
      nicht implementiert ist.
    - Fehler fuer unsupported Instructions behalten, bis die jeweilige Semantik
      wirklich implementiert ist.

Tests:
    - Regressionstest fuer Float-Comparison-Codegen.
    - Regressionstest fuer ungueltige Char-Literals ohne Panic.
    - Regressionstest fuer mehrere returns in einem Block.
    - Regressionstest fuer void-Parameter.
    - Pipeline-Tests fuer einfache Programme:
        return int
        let + assignment
        arithmetic
        if/else
        while
        function call
    - CLI-Ende-zu-Ende-Test fuer ein ausfuehrbares Programm.

3. Danach / groessere Sprachfeatures
============================================================

Strings:
    - String als echtes Struct definieren, z.B. ptr + len.
    - String-Literals in Datenbereich ablegen.
    - IR fuer const.string oder Struct-Init festlegen.
    - Assembly-Ausgabe fuer String-Werte implementieren.

Arrays / Listen:
    - Array/List-Typen definieren.
    - List-Literals lowern.
    - GetElementAddr mit Elementgroesse implementieren.
    - Index-Zugriff und Index-Assignment.

Structs:
    - Top-Level Structs in IrModule.types erzeugen.
    - Struct-Literals lowern.
    - InitStruct in Assembly implementieren.
    - FieldAccess und Field-Assignment lowern.
    - GetFieldAddr mit Offsetberechnung implementieren.
    - Methoden als normale Funktionen lowern.

Variants:
    - Top-Level Variants in IrModule.types erzeugen.
    - Variant-Literals lowern.
    - InitVariant in Assembly implementieren.
    - Payload-Variants spaeter einplanen.
    - Pattern/Match erst definieren, bevor Codegen gebaut wird.

Generics:
    - Vor Low-IR monomorphisieren.
    - Konkrete Named Types erzeugen, z.B. List_i64.
    - Name-Mangling fuer generische Instanzen festlegen.

Traits / Operator Overloading:
    - Trait-Implementierungen pruefen.
    - Trait-Calls zu konkreten Calls aufloesen.
    - Operatoren fuer primitive Typen direkt lowern.
    - Operatoren fuer eigene Typen als Funktions-/Trait-Calls lowern.
    - Funktion Overloading nur mit klarer Signaturauflösung starten.

Inline Assembly:
    - Aktuell als Blackbox behalten.
    - Spaeter Inputs/Outputs/Clobbers definieren.
    - Variablenreferenzen in inline asm nur mit explizitem Constraint-Modell.

Linking / Runtime:
    - Klaeren, ob eigener Assembler/Linker wirklich Ziel ist.
    - Bis dahin vorhandene Systemtools nutzen.
    - Minimal-Runtime fuer I/O erst nach stabilem Calling-Modell bauen.
