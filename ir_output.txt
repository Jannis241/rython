---------- IrModule ----------

IrModule ist das Top-Level-Objekt, das rython_to_ir an ir_to_assembly
uebergibt.

Felder:

- inline_assembly: Vec<String>
    Top-level Inline-Assembly als rohe Strings.

- functions: Vec<IrFunction>
    Alle Funktionen des Programms.

- globals: Vec<IrGlobal>
    Globale Variablen.

- constants: Vec<IrConstant>
    Top-level Konstanten.

- types: Vec<IrTypeDefinition>
    Eigene Typdefinitionen, also Structs und Variants.

Aktueller Stand der erzeugten Module:

- Funktionen werden in functions abgelegt.
- Top-level asm wird in inline_assembly abgelegt.
- globals, constants und types sind Teil der externen IR-Struktur, koennen aber
  aktuell leer bleiben.


---------- IrValue ----------

IrValue beschreibt konkrete Werte, die direkt in der Modulstruktur stehen
koennen, z.B. als Initialwert einer globalen Variable oder Konstante.

Varianten:

- Primitive(PrimitiveValue)
    Ein primitiver Wert: int, float, bool oder char.

- String(String)
    Ein String-Wert.

- Struct { fields: Vec<(String, IrValue)> }
    Ein Struct-Wert. Jedes Feld besteht aus Feldname und Feldwert.

- Array(Vec<IrValue>)
    Ein Array-Wert mit den enthaltenen Elementwerten.

- Variant { case_name: String }
    Ein Variant-Wert mit dem ausgewaehlten Case.

- Null
    Ein Null-Wert, z.B. fuer Pointer oder fehlende Adresse.

Beispiele:

IrValue::Primitive(PrimitiveValue::Int(3))
IrValue::String("hello".to_string())
IrValue::Struct {
    fields: vec![
        ("x".to_string(), IrValue::Primitive(PrimitiveValue::Int(1))),
        ("y".to_string(), IrValue::Primitive(PrimitiveValue::Int(2))),
    ],
}
IrValue::Null


---------- IrGlobal ----------

IrGlobal beschreibt eine globale Variable.

Felder:

- name: String
    Name der globalen Variable.

- ty: IrType
    Typ der globalen Variable.

- value: IrValue
    Initialer Wert der globalen Variable.


---------- IrConstant ----------

IrConstant beschreibt eine Top-level Konstante.

Felder:

- name: String
    Name der Konstante.

- ty: IrType
    Typ der Konstante.

- value: IrValue
    Wert der Konstante.


---------- IrField ----------

IrField verbindet einen Namen mit einem Typ. Es enthaelt keinen Wert.

Felder:

- name: String
- ty: IrType

IrField wird z.B. fuer Struct-Felder und Funktionsparameter benutzt.

Beispiel:

IrField {
    name: "a",
    ty: IrType::I64,
}

Das bedeutet: Es gibt ein Feld oder einen Parameter namens "a" mit dem Typ I64.
Der konkrete Laufzeitwert steht nicht in IrField.


---------- IrTypeDefinition ----------

IrTypeDefinition beschreibt eigene Typen neben den primitiven IR-Typen.

Varianten:

- Struct { name: String, fields: Vec<IrField> }
    Definiert einen Struct-Typ.

    Beispiel:

    IrTypeDefinition::Struct {
        name: "Point",
        fields: vec![
            IrField { name: "x", ty: IrType::I64 },
            IrField { name: "y", ty: IrType::I64 },
        ],
    }

- Variant { name: String, cases: Vec<String> }
    Definiert einen Variant/Enum-aehnlichen Typ.

    Beispiel:

    IrTypeDefinition::Variant {
        name: "Option",
        cases: vec!["Some", "None"],
    }


---------- IrFunction ----------

IrFunction beschreibt eine einzelne Funktion in der IR.

Felder:

- name: String
    Name der Funktion.

- parameter: Vec<IrField>
    Parameter der Funktion. Jeder Parameter hat einen Namen und einen Typ,
    aber keinen konkreten Wert.

- return_type: IrType
    Rueckgabetyp laut Funktionssignatur.

- blocks: Vec<IrBlock>
    Code der Funktion, aufgeteilt in Basic Blocks.

Beispiel:

fn main(a: int, b: int) int {
    ...
}

wird ungefaehr:

IrFunction {
    name: "main",
    parameter: vec![
        IrField { name: "a", ty: IrType::I64 },
        IrField { name: "b", ty: IrType::I64 },
    ],
    return_type: IrType::I64,
    blocks: vec![...],
}


---------- IrBlock ----------

IrBlock ist ein Basic Block: eine lineare Folge von Instructions plus ein
Terminator am Ende.

Felder:

- label: String
    Name des Blocks, z.B. "entry:".

- instructions: Vec<IrInstruction>
    Normale Instructions, die nacheinander ausgefuehrt werden.

- terminator: Terminator
    Abschluss des Blocks, z.B. return, jump oder branch.

Beispiel:

IrBlock {
    label: "entry:",
    instructions: vec![
        IrInstruction::PrimitiveConst { ... },
        IrInstruction::Alloca { ... },
        IrInstruction::Store { ... },
    ],
    terminator: Terminator::Ret(None),
}


---------- TempId ----------

TempId ist eine temporaere ID innerhalb einer Funktion.

Viele Instructions schreiben ihr Ergebnis in eine TempId. Andere Instructions
lesen spaeter aus diesen TempIds.

Beispiel:

PrimitiveConst { temp_id: TempId(0), ... }

bedeutet: Der konstante Wert liegt danach unter TempId(0).

Eine TempId kann je nach Instruction einen Wert oder eine Adresse bezeichnen.
Bei Alloca, GlobalAddr, GetFieldAddr und GetElementAddr bezeichnet die Ergebnis-
TempId eine Adresse. Load liest aus einer solchen Adresse, Store schreibt in eine
solche Adresse.


---------- IrInstruction ----------

IrInstruction beschreibt die eigentlichen Operationen im Code. Aus diesen
Instructions baut ir_to_assembly spaeter Assembly.

Aktuelle Varianten:

- PrimitiveConst { temp_id: TempId, ty: IrType, value: PrimitiveValue }
    Erzeugt einen primitiven konstanten Wert.
    Das Ergebnis liegt danach in temp_id.

- LoadParam { temp_id: TempId, index: usize, ty: IrType }
    Laedt den Funktionsparameter mit 0-basiertem Index.
    Beispiel: index 0 ist der erste Parameter.
    Das Ergebnis liegt danach in temp_id.

- Alloca { temp_id: TempId, ty: IrType }
    Reserviert Speicher fuer einen Wert vom Typ ty.
    temp_id bezeichnet danach die Adresse dieses Speicherplatzes.

- Load { temp_id: TempId, ty: IrType, addr: TempId }
    Liest einen Wert vom Typ ty aus der Adresse addr.
    Das Ergebnis liegt danach in temp_id.

- Store { ty: IrType, value: TempId, addr: TempId }
    Schreibt den Wert value in die Adresse addr.
    Store hat keine Ergebnis-TempId.

- Binary { temp_id: TempId, ty: IrType, op: IrBinaryOp, lhs: TempId, rhs: TempId }
    Fuehrt eine binaere Operation mit lhs und rhs aus.
    Das Ergebnis liegt danach in temp_id.

- Unary { temp_id: TempId, ty: IrType, op: IrUnaryOp, value: TempId }
    Fuehrt eine unaere Operation auf value aus.
    Das Ergebnis liegt danach in temp_id.

- Call {
      temp_id: Option<TempId>,
      function_name: String,
      args: Vec<TempId>,
      return_type: IrType,
  }
    Ruft eine Funktion auf.
    args sind die bereits berechneten Argumentwerte als TempIds.
    temp_id ist Some(...) bei einem Rueckgabewert und None bei void.

- GlobalAddr { temp_id: TempId, name: String, ty: IrType }
    Liefert die Adresse einer globalen Variable.
    temp_id bezeichnet danach diese Adresse.

- Asm { code: String }
    Inline-Assembly innerhalb einer Funktion.

- InitVariant { temp_id: TempId, ty: IrType, case_name: String }
    Erzeugt einen Variant-Wert mit einem bestimmten Case.
    ty ist normalerweise ein benannter Typ, z.B. IrType::Named("Option").

- InitArray {
      temp_id: TempId,
      element_type: IrType,
      elements: Vec<TempId>,
  }
    Erzeugt einen Array-Wert.
    elements sind die TempIds der einzelnen Elementwerte.

- GetElementAddr { temp_id: TempId, base_addr: TempId, index: TempId }
    Liefert die Adresse eines Array-Elements.
    base_addr ist die Adresse des Arrays.
    index ist ein Wert-Temp fuer den Index.
    temp_id bezeichnet danach die Elementadresse.

- InitStruct {
      temp_id: TempId,
      ty: IrType,
      fields: Vec<(String, TempId)>,
  }
    Erzeugt einen Struct-Wert.
    fields enthaelt pro Feld den Feldnamen und die TempId des berechneten
    Feldwerts.

- GetFieldAddr { temp_id: TempId, base_addr: TempId, field_name: String }
    Liefert die Adresse eines Feldes innerhalb eines Structs.
    base_addr ist die Adresse des Structs.
    temp_id bezeichnet danach die Feldadresse.


---------- IrBinaryOp ----------

Operationen fuer IrInstruction::Binary:

- Add
- Sub
- Mul
- Div
- Mod
- Eq
- Ne
- Lt
- Le
- Gt
- Ge
- And
- Or
- BitAnd
- BitOr
- BitXor
- Shl
- Shr


---------- IrUnaryOp ----------

Operationen fuer IrInstruction::Unary:

- Neg
- Not
- BitNot


---------- Terminator ----------

Ein Terminator beendet einen IrBlock.

Varianten:

- Ret(Option<TempId>)
    Return.
    Ret(Some(temp)) gibt den Wert temp zurueck.
    Ret(None) ist return void.

- Jump { target: String }
    Unbedingter Sprung zu einem anderen Block.

- Branch { condition: TempId, then_block: String, else_block: String }
    Bedingter Sprung.
    Wenn condition true bzw. ungleich 0 ist, geht es zu then_block,
    sonst zu else_block.


---------- PrimitiveValue ----------

PrimitiveValue haelt konkrete primitive Werte.

Varianten:

- Int(i64)
- Float(f64)
- Bool(bool)
- Char(char)

Beispiele:

PrimitiveValue::Int(3)
PrimitiveValue::Float(3.5)
PrimitiveValue::Bool(true)
PrimitiveValue::Char('a')


---------- IrType ----------

IrType beschreibt den Typ eines Werts oder Speicherplatzes, ohne den konkreten
Wert selbst zu enthalten.

Varianten:

- I64
    64-bit Integer.

- Bool
    Boolean.

- Void
    Kein Rueckgabewert / kein Wert.

- F64
    64-bit Float.

- Named(String)
    Benannter Typ fuer eigene Typen oder spezielle Typen wie "string".

- Pointer(Box<IrType>)
    Pointer auf einen anderen IR-Typ.
    Der innere Typ beschreibt, worauf der Pointer zeigt.

Beispiele:

IrType::I64
IrType::F64
IrType::Bool
IrType::Void
IrType::Named("string".to_string())
IrType::Named("Point".to_string())
IrType::Pointer(Box::new(IrType::I64))
